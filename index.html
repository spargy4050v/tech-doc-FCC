Introduction

Overview

Running Ubuntu in a virtualized environment on Windows allows seamless integration of Linux functionalities alongside Windows. This setup benefits developers, testers, and users needing Linux tools without dual-boot complexities. VirtualBox provides secure, isolated environments for software testing, development, and experimentation. It optimizes CPU, memory, and disk usage, supports snapshots for easy recovery, and facilitates efficient resource management.

Virtualization simplifies application testing and Linux learning by providing isolated environments for experimentation without risking the main OS. It supports compatibility testing, snapshots for quick rollback, and efficient resource management. Virtual machines also offer hands-on experience with Linux commands and system administration, making them valuable for both learning and development purposes.

Prerequisites

Before installing Ubuntu in a virtualized environment on Windows using VirtualBox, ensure your system meets the following requirements:

  1. Hardware Requirements:
    • Processor: Intel VT-x or AMD-V virtualization hardware extensions should be enabled in your computer's BIOS settings.
    • Memory: Allocate sufficient RAM for both Windows and Ubuntu. Typically, 4GB or more is recommended for a smooth experience, depending on your workload.
    • Disk Space: Ensure you have ample free disk space for installing VirtualBox, Ubuntu, and any additional software or files you plan to work with.
  2. Software Requirements:
    • Virtualization Software: Download and install Oracle VM VirtualBox, the recommended virtualization software for running Ubuntu on Windows. Ensure you have the latest version compatible with your Windows operating system.
    • Ubuntu ISO Image: Download the Ubuntu Desktop ISO file from the official Ubuntu website (https://ubuntu.com/download/desktop). Choose the appropriate version, such as the LTS (Long Term Support) release, based on your needs.

Meeting these prerequisites ensures a smooth setup of Virtualized Ubuntu on your Windows system, facilitating efficient software testing, development, and learning experiences.

Setting Up VirtualBox

To set up VirtualBox on your Windows system for running Ubuntu, follow these essential steps:

  1. Installing VirtualBox:
    • Run the installer and follow the on-screen instructions to complete the installation.
    • Accept any prompts that require administrative privileges during the installation procedure.
  2. Configure VirtualBox
    • Launch the VirtualBox from the your Desktop.
    • Familiarize yourself with the VirtualBox interface and settings
  3. Optional: Virtualization Support
    • If needed, enable virtualization support (VT-x/AMD-V) in your computer's BIOS settings for optimal performance.

With VirtualBox installed and configured, you're ready to proceed to create a virtual machine (VM) and install Ubuntu. Refer to the next steps in your documentation for detailed guidance on setting up your Ubuntu VM within VirtualBox.

Creating a Virtual Machine

To create a virtual machine (VM) in VirtualBox for running Ubuntu on Windows, follow these steps:

  1. Open VirtualBox: Launch VirtualBox form your desktop or Start menu.
  2. Click on "New": In the VirtualBox window, click the "New" button to start creating a new VM.
  3. Name and Operating System: Enter a name for your VM (e.g., "Ubuntu") and select the operating system type and version (e.g., Linux and Ubuntu).
  4. Allocate Memory: Choose the amount of RAM to allocate to the VM. It's recommended to allocate at least 2GB for Ubuntu, depending on your system's resources.
  5. Create a Virtual Hard Disk: Create a new virtual hard disk for the VM. Select the disk file type and choose dynamically allocated or fixed size depending on your preference.
  6. Specify Disk File Location and Size: Set the location and size of the virtual disk. Allocate sufficient space for Ubuntu and any applications you plan to install.
  7. Finish and Create: Click "Create" to finish creating the VM. VirtualBox will now create the VM with the specified settings.

Once the VM is created, you can proceed to install Ubuntu on the virtual machine. Refer to your Ubuntu installatin guide within VirtualBox for detailed steps on installing the operating system and configuring the VM for optimal performance.

Installing Ubuntu

To install Ubuntu in VirtualBox on your Windows system, follow these steps:

  1. Start the VM:
    • With the Ubuntu ISO attached, start your VM by clicking "Start" in VirtualBox.
  2. Install Ubuntu:
    • The VM will boot from the Ubuntu ISO. Follow the on-screen instructions to install Ubuntu>
    • Choose your language, keyboard layout, and installation type (e.g., Erase disk and install Ubuntu or Something else for custom partitioning).
    • Create a user account and password when prompted.
  3. Complete Installation:
    • Once installation completes, restart the VM. Remove the Ubuntu ISO from the VM's virtual disk drive in VirtualBox settings.
  4. Install VirtualBox Guest Additions: (Optional but recommended):
    • In the running Ubuntu VM window, click on "Devices" > "Insert Guest Additions CD image...".
    • Open a terminal (Ctrl+Alt+T) in Ubuntu and run the command given below:
      • sudo sh /media/<Username>/VBox_GAs_<version>/VBoxLinuxAdditions.run
    • Restart the VM after installation completes for enhanced performance and features.

By following these steps, you can install Ubuntu in VirtualBox on your Windows system and begin using Ubuntu alongside your existing operating system for various tasks and projects.

Git Commands

Here are some essential Git commands commonly used for version control and collaboration in software development

  1. Initialize a Git Repository:
    2. git init

    Initialize a new Git repository in the current directory.

  2. Clone a Repository:
    3. git clone <repository_url>

    Clone a remote repository from the specified URl to your local machine.

  3. Add Files to Staging Area:
    4. git add <file_name>

    Add specified file(s) to the staging area to be included in the next commit.

  4. Commit Changes:
    5. git commit "Commit message"

    Record changes to the repository with a descriptive commit message.

  5. Push Changes to Remote Repository:
    6. git push origin <branch_name>

    Upload local commits to the remote repository specified by origin <branch_name>

  6. Pull Latest Changes from Remote Repository
    7. git pull origin <branch_name>

    Fetch and merge changes from the remote repository into the current branch.

  7. Check Status of Working Tree:
    8. git status

    Display the current state of the working directory and staging area.

  8. View Commit History
    9. git log

    Display a list of commits in reverse chronological order.

  9. Create a New Branch:
    10. git branch <branch_name>

    Create a new branch based on the current HEAD

  10. Switch Branches:
    11. git checkout <branch_name>

    Switch to the specified branch and update the working directory.

These commands from the core workflow for using Git to manage version, collaborate with others, and maintain project history effectively. Adjust parameters and options based on your specific project needs and Git repository setup.
